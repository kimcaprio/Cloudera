 클라우데라 머신 러닝과 클라우데라 데이터 사이언스 워크벤치의 주요 차이점 이 주제에서는 클라우데라 데이터 사이언스 워크벤치와 클라우드 네이티브 대응 솔루션인 클라우데라 머신 러닝의 몇 가지 주요 차이점을 중점적으로 설명합니다. 클라우데라 머신 러닝(CML)은 클라우데라 데이터 사이언스 워크벤치(CDSW)와 어떤 관련이 있나요? CML은 신속한 프로비저닝, 탄력적인 자동 확장, 분산 종속성 격리, 분산 GPU 트레이닝과 같은 클라우드 네이티브 이점을 통해 Cloudera Data Science Workbench(CDSW)의 엔드투엔드 워크플로우를 확장합니다. 스케일아웃 컴퓨팅을 위한 별도의 CDH 클러스터 없이도 자체적인 네이티브 분산 컴퓨팅 워크로드를 실행할 수 있습니다. 기존 Kubernetes 환경의 CDP에서 실행되도록 설계되어 일부 고객의 운영 비용을 절감하는 동시에 멀티클라우드 이동성을 제공합니다. 퍼블릭 클라우드의 경우, 관리형 클라우드 Kubernetes 서비스에는 EKS, AKS 또는 GKE가 포함되며, 프라이빗 클라우드의 경우 Red Hat OpenShift 또는 ECS(임베디드 컨테이너 서비스)가 포함됩니다. 두 제품 모두 강력한 보안과 거버넌스를 통해 데이터 엔지니어와 데이터 과학 팀이 공유 데이터와 컴퓨팅에서 생산성을 높일 수 있도록 지원합니다. 두 제품은 광범위한 코드를 공유합니다. 한 가지 주요 차이점이 있습니다: CDSW는 게이트웨이 노드에서 실행되고 분산된 컴퓨팅 워크로드를 클러스터로 푸시하여 기존 CDH 클러스터를 확장합니다. CDSW는 아파치 스파크를 포함한 분산 컴퓨팅을 위해 단일 CDH 클러스터가 필요하고 이를 지원합니다. 이와는 대조적으로, CML은 독립형이며 자체 분산 컴퓨팅을 관리하며, 기본적으로 Kubernetes의 컨테이너에서 Apache Spark를 포함한 워크로드를 실행합니다(이에 국한되지 않음). 참고: 기존 클러스터에 연결하여 분산 컴퓨팅, 데이터 또는 메타데이터(SDX)를 활용할 수 있습니다. 표 1. 주요 차이점 CDSW CML 아키텍처 CDSW는 CDP-DC, CDH(5 또는 6), HDP 클러스터에서 실행할 수 있으며, 클러스터의 하나 이상의 전용 게이트웨이 노드에서 실행됩니다. CML은 독립형이며 연결된 CDH/HDP 클러스터가 필요하지 않습니다. 1개의 마스터 호스트와 다수의 작업자 호스트 개념. 지정된 마스터 및 작업자 호스트가 없으며 모든 노드가 임시 노드입니다. CDH/HDP 클러스터를 통해 통합된 보안 케르베로스 인증 클라우데라 데이터 플랫폼(CDP)을 통해 FreeIPA를 사용한 중앙 집중식 ID 관리. LDAP/SAML을 통한 외부 인증. 앱 스토리지 프로젝트 파일, 내부 포스트그레스DB, 라이브 로그는 모두 마스터 호스트에 영구적으로 저장됩니다. 필요한 모든 영구 저장소는 클라우드 관리형 블록 스토어, NFS, 관계형 데이터 스토어에 저장됩니다. 예를 들어, AWS의 경우 EFS를 통해 관리됩니다. Python/R/Scala 워크로드 계산은 클러스터의 CDSW 게이트웨이 노드에서 실행됩니다. Python/R/Scala 워크로드는 CDP/클라우드 제공자가 관리하는 K8s 클러스터에서 실행됩니다. CDSW는 Spark-on-YARN과 같은 분산 컴퓨팅 워크로드를 CDH/HDP 클러스터로 푸시합니다. Spark-on-YARN은 지원되지 않으며, 대신 Spark-on-K8이 지원됩니다. 워크로드는 고객 환경 내에 프로비저닝된 전용 K8s 클러스터에서 실행됩니다. 자동 확장 없음. 클라우드 서비스 공급자를 통한 자동 확장. 수요에 따라 클러스터 크기를 확장/축소하기 위해 Kubernetes/노드 수준 오토스케일링이 사용됩니다. 패키징 다운로드 가능한 RPM 및 CSD로 제공됩니다. CDP에서 관리형 서비스로 제공됩니다. Spark는 CDH와 함께 패키징됩니다. K8의 Spark는 CML과 함께 패키징되어 외부 클러스터에 대한 종속성이 없습니다. 데이터 액세스 데이터는 일반적으로 연결된 CDH/HDP 클러스터의 HDFS, Hive, HBase 등에 상주합니다. 데이터는 S3와 같은 오브젝트 스토리지 또는 CDP에 등록된 기존 워크로드 클러스터에 있을 수 있습니다.
