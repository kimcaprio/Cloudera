머신 러닝 및 기타 모델을 프로덕션 환경에 배포할 수 있는 ML Ops CML. CML을 통해 사용자는 머신 러닝 및 기타 모델을 작업 기능을 통한 배치 프로세스로 프로덕션에 배포하거나 모델 기능을 사용하여 거의 실시간에 가까운 REST API로 배포할 수 있습니다. 또한 CML은 프로덕션 환경에서 이러한 모델을 유지 관리, 모니터링 및 관리하는 데 도움이 되는 다양한 기능을 제공합니다. 모델 거버넌스 기능은 배포된 모든 모델을 Cloudera 데이터 카탈로그에서 추적하고, 사용자가 모델-데이터 계보를 제공하기 위해 모델 학습에 사용된 데이터 테이블을 지정할 수 있게 해줍니다. 배포된 모델에는 요청 처리량, 지연 시간, 리소스 소비량 등 배포된 CML 모델과 관련된 기술 메트릭을 모니터링할 수 있는 대시보드가 내장되어 있습니다. 또한 사용자는 각 추론 이벤트와 관련된 임의의 비즈니스 지표를 추적하고, 자동으로 생성된 UUID를 사용하여 결과를 데이터 웨어하우스 또는 기타 소스의 지연 지표와 일치시킬 수 있습니다. 이러한 메트릭을 분석함으로써 사용자는 정확도와 같은 집계된 메트릭에 대해 지속적으로 모델을 평가할 수 있습니다. 핵심 기능 이 섹션에서는 클라우데라 머신 러닝의 핵심 기능에 대해 자세히 설명합니다. 클라우데라 머신 러닝은 엔드투엔드 머신 러닝 워크플로우를 다루며, 원활한 분산 종속성 관리를 통해 스케일아웃 데이터 엔지니어링과 머신 러닝을 위해 Python, R, Spark-on-Kubernetes를 포함한 완전히 격리되고 컨테이너화된 워크로드를 지원합니다. 세션을 통해 데이터 과학자는 작업 공간 전체에서 사용 가능한 CPU, 메모리, GPU 컴퓨팅을 직접 활용하는 동시에 데이터 레이크의 데이터에 직접 연결할 수 있습니다. 실험을 통해 데이터 과학자는 여러 가지 변형된 모델 훈련 워크로드를 실행하고 각 실험의 결과를 추적하여 최상의 모델을 훈련할 수 있습니다. 클릭 몇 번으로 모델을 배포할 수 있으므로 프로덕션의 장애물을 제거할 수 있습니다. 이러한 모델은 고가용성 방식으로 REST 엔드포인트로 제공되며, 자동화된 계보 구축 및 MLOps 목적의 지표 추적이 가능합니다. 잡을 사용하여 모델 드리프트 모니터링, 필요에 따라 모델 재교육 및 재배포를 자동으로 시작하는 등 전체 엔드투엔드 자동화된 파이프라인을 오케스트레이션할 수 있습니다. 애플리케이션은 클릭 몇 번으로 비즈니스 사용자에게 인터랙티브한 경험을 제공합니다. 이러한 애플리케이션 개발에는 플라스크나 샤이니와 같은 프레임워크가 사용될 수 있으며, 이러한 경험을 구축하기 위한 포인트 앤 클릭 인터페이스로서 클라우데라 데이터 시각화도 제공됩니다.
