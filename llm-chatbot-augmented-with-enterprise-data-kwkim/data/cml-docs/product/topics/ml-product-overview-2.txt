 ML 워크스페이스는 유연하고 확장 가능한 엔진을 통해 Python, R, Scala 및 Spark 워크로드의 완전 컨테이너화된 실행을 지원합니다. AI 애플리케이션 분석 애플리케이션은 CML 프로젝트 내에서 장기간 실행되는 애플리케이션을 호스팅할 수 있는 공간을 제공합니다. CML이 데이터 과학자가 고급 분석과 모델을 프로덕션 환경에서 수행할 수 있는 공간을 제공한다면, 분석 애플리케이션은 CML 프로젝트 내에서 장기적으로 실행되는 애플리케이션을 호스팅할 수 있는 공간을 제공합니다. 이를 통해 더 많은 사용자 그룹이 CML에서 발견한 인사이트를 활용할 수 있습니다. 애플리케이션은 Flask 및 Streamlit과 같은 다양한 프레임워크로 구축할 수 있습니다. 이러한 애플리케이션은 자체적으로 격리된 컴퓨팅 인스턴스 내에서 실행되므로 시간 초과를 방지하고 ML 런타임을 활용할 수 있습니다. 사용자는 웹을 통해 애플리케이션에 액세스할 수 있습니다. 애플리케이션은 대화형 데이터 시각화를 호스팅하거나 CML에서 배포된 모드를 위한 UI 프런트엔드를 제공하는 등 다양한 사용 사례에 사용될 수 있습니다. 탐색적 데이터 과학 CML을 사용하면 데이터 실무자가 단일 사용자 인터페이스 내에서 데이터 집합을 검색, 쿼리, 쉽게 시각화할 수 있습니다. 모든 데이터 과학 프로젝트의 시작은 필요한 데이터를 찾고 이해하는 것에서 시작됩니다. CML은 탐색적 데이터 분석에 필요한 모든 도구를 단일 UI에 통합하여 데이터 실무자가 애플리케이션 간에 이동할 필요가 없고, IT 부서가 여러 도구를 지원할 필요가 없습니다. CML은 사용자가 연결할 수 있는 사용 가능한 데이터 자산의 보기, 해당 데이터 원본을 쿼리할 수 있는 SQL 편집기, 데이터를 이해하고 인사이트를 전달할 수 있는 간편한 드래그 앤 드롭 방식의 시각화 도구를 제공합니다.
