ML 런타임클라우데라 문서 ML 런타임 ML 런타임은 데이터 과학 워크로드를 실행하고 기본 클러스터에 대한 액세스를 중개하는 역할을 담당합니다.  CML을 사용하면 대화형 세션, 예약된 작업, 배포된 모델 또는 애플리케이션을 통해 모든 코드를 실행할 수 있습니다. 데이터 과학자는 대화형 세션을 사용해 데이터를 탐색하거나 모델을 개발할 수 있습니다. 작업을 만들어 지정된 시간에 실행되도록 예약하거나, 작업을 모델로서 프로덕션화하여 REST 엔드포인트를 제공하거나, 비즈니스 사용자를 위한 대화형 데이터 대시보드를 제공하는 애플리케이션으로 만들 수 있습니다. 이러한 모든 워크로드는 쿠버네티스 상단의 ML 런타임 컨테이너 내에서 실행됩니다.  클라우데라 ML 런타임은 특정 사용 사례를 지원하도록 설계되었습니다. 단일 편집기(예: Workbench, Jupyterlab)와 함께 제공되며, 단일 언어 커널(예: Python 3.8 또는 R 4.0)이 제공되고, UNIX 도구 및 유틸리티 또는 언어 라이브러리 및 패키지 세트가 포함되어 있습니다.  ML 런타임은 오픈 소스로 제공되며, cloudera/ml-runtimes GitHub 리포지토리에서 확인할 수 있습니다. 런타임 환경을 완전히 이해해야 하거나 런타임을 처음부터 새로 빌드하려는 경우, 이 리포지토리에서 ML 런타임 컨테이너 이미지를 빌드하는 데 사용된 Docker파일에 액세스할 수 있습니다. 대부분의 데이터 과학 사용 사례를 포괄하는 다양한 런타임이 기본으로 지원되지만, 특별한 요구 사항은 사용자 지정 ML 런타임 컨테이너 이미지를 빌드하여 충족할 수 있습니다. 또한 CML은 CPU, GPU, 메모리에 대한 쿼터 관리를 지원하여 사용자가 CML 작업 영역 내에서 액세스할 수 있는 리소스의 양을 제한할 수 있습니다.  ML 런타임 이전에는 레거시 엔진을 통해 유사한 기능을 제공했습니다. 더 이상 사용되지 않는 이러한 컨테이너 이미지는 모놀리식 아키텍처, 모든 커널(Python, R, Scala), 유용해 보이는 모든 패키지 및 라이브러리가 이미지에 포함되었습니다.   상위 주제: 아키텍처 개요
