Kubernetes의 Spark클라우데라 문서 Kubernetes의 Spark클라우데라 머신 러닝(CML)에서는 런타임 애드온을 통해 여러 Spark 버전을 사용할 수 있습니다. 데이터 과학자는 모든 워크로드에 대해 구성할 Spark 버전을 선택할 수 있습니다. CML은 런타임 컨테이너를 구성하고 모든 종속 요소를 마운트합니다. CML은 Kubernetes 클러스터 백엔드에 대한 Spark의 지원을 통해 Spark 워크로드의 완전 컨테이너화된 실행을 지원합니다. 사용자는 대화형 및 배치 모드에서 모두 Spark와 상호 작용할 수 있습니다. 배치 모드와 대화형 모드 모두에서, Spark 실행자를 포함한 종속성 관리는 CML과 Kubernetes에 의해 투명하게 관리됩니다. 추가 구성이 필요하지 않습니다. 대화형 모드에서 CML은 확장 가능한 프로젝트 스토리지를 위해 클라우드 제공업체를 활용하고, 배치 모드에서는 컨테이너 이미지를 통해 종속성을 관리합니다. 또한 CML은 Kubernetes를 통한 네이티브 클라우드 자동 확장을 지원합니다. 클러스터에 워크로드를 실행하는 데 필요한 용량이 부족하면 자동으로 추가 노드를 확장할 수 있습니다.          관리자는 자동 확장 상한을 구성하여 컴퓨팅 클러스터가 성장할 수 있는 규모를 결정할 수 있습니다. 클러스터 크기가 커질수록 컴퓨팅 비용이 증가하므로, 상한을 구성할 수 있는 방법이 있으면 관리자는 예산 범위 내에서 관리할 수 있습니다. 자동 확장 정책은 GPU 노드와 같은 이기종 노드 유형도 고려할 수 있습니다. CML에서 각 프로젝트는 사용자 또는 팀이 소유합니다. 사용자는 프로젝트에서 여러 세션을 시작할 수 있습니다. 워크로드는 각 사용자에 대해 별도의 쿠버네티스 네임스페이스 내에서 시작되므로 쿠버네티스 수준에서 사용자 간 격리가 보장된다.   상위 주제: 아키텍처 개요
