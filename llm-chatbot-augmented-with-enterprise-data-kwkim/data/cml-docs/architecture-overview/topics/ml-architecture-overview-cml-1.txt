CML 아키텍처클라우데라 문서 CML 아키텍처 CML 작업 공간이 프로비저닝되면, 엔드투엔드 머신 러닝 워크플로우를 위해 Cloudera 머신 러닝(CML)을 사용할 수 있습니다.  CML은 프레젠테이션 티어, 애플리케이션 티어, 데이터 티어로 구성된 3티어 애플리케이션입니다.  웹 티어 CML은 데이터 과학자를 위한 워크로드 및 리소스 관리 작업을 간소화하는 UI를 제공하는 웹 애플리케이션입니다. 사용자가 분석 파이프라인을 배포 및 확장하고 안전한 구획화된 환경에서 동료와 협업할 수 있는 편리한 방법을 제공합니다.  CML은 HTTPS, 웹소켓, gRPC를 사용해 통신합니다. 외부 통신은 웹 UI와 API를 위한 HTTP와 Websocket으로 제한됩니다. 클러스터 내 서비스 간 통신은 gRPC를 사용하며, TLS(전송 계층 보안)를 사용하여 암호화되고 상호 인증됩니다.   애플리케이션 티어 애플리케이션 티어는 사용자가 실행 중인 실제 워크로드 집합을 사용합니다. 이러한 워크로드는 쿠버네티스 네임스페이스에 격리되어 있으며 특별히 표시된 컴퓨팅 노드에서 실행됩니다. 사용자 수요에 따라 클러스터 크기를 확장/축소하기 위해 Kubernetes/노드 수준 자동 확장이 사용됩니다. 사용자 코드는 관리 서버, 빌드 대기열, Docker용 이미지를 제공하는 레지스트리, git 서버, 이미지 빌드를 실제로 수행하는 빌더를 포함하는 일련의 소스-투-이미지 포드(S2I)를 통해 Docker 이미지로 베이크됩니다.             기존에는 이러한 이미지가 호스트 머신의 Docker를 사용했지만, 일부 플랫폼에서 보안 및 규정 준수를 위해 CML은 컨테이너 내 빌드로 전환했습니다.   데이터 티어 CML은 세션, 작업, 모델 및 애플리케이션과 같은 사용자 워크로드의 메타데이터를 지속하기 위해 내부 Postgres 데이터베이스를 사용하며, 이는 포드로 실행되고 클라우드 제공업체의 블록 스토리지 제품(예: AWS의 경우 EBS, Azure의 경우 Premium_LRS)을 사용하여 영구 볼륨으로 백업됩니다.
