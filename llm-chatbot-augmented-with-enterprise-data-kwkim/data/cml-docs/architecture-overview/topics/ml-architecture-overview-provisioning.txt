프로비저닝클라우데라 문서 프로비저닝 CML은 CDP 컨트롤 플레인을 활용하여 데이터 서비스를 관리하므로 CML 워크스페이스를 프로비저닝하고 삭제할 수 있습니다. CDP 컨트롤 플레인은 클라우드 네이티브 기능을 활용하여 인프라를 제공하기 위해 클라우드 관리형 Kubernetes(K8)와 함께 CPU, 메모리, GPU 리소스에 동적으로 액세스합니다.  프로비저닝 중에 머신 러닝 애플리케이션은 CDP ID 공급자를 통해 서비스의 최종 사용자(ML)를 인증하도록 구성되며, 이 인증은 고객의 ID 공급자로 다시 '체인'됩니다. 결과적으로 머신 러닝 프로비저닝 인스턴스는 원활한 고객 SSO를 허용합니다.  CML 워크스페이스를 프로비저닝하면 다음과 같은 일이 발생합니다:     CDP 컨트롤 플레인은 클라우드 환경에서 다음을 수행합니다:  cloudera.site 도메인으로 TLS 인증서 및 도메인 이름 요청 SSO 구성 식별 환경에 대한 SDX 구성 식별 CDP 컨트롤 플레인이 관리형 Kubernetes 클러스터 프로비저닝 CDP 컨트롤 플레인이 Kubernetes 환경에 CML 설치 스토리지가 관리형 서비스 공급자를 통해 직접 마운트됨 CML은 클라우드 공급자 로드 밸런서 및 네트워킹 인프라를 사용하여 리소스를 파티션합니다. 또한 CML은 고객이 자동 확장을 지정할 수 있도록 클라우드 제공업체 인프라를 활용합니다.  CML은 DNS와 인증서를 프로비저닝합니다. CML은 고객을 위해 지속적으로 인증서를 갱신합니다.   상위 주제: 아키텍처 개요
